// Storage row (rows M-N, at the bottom of the canvas): 18 dedicated containers, one per edge in acu_dependency_tree.html — no manifold splitting, matching the tree's own design principle ("no node splits its output... unless" the Stator 5/5 case). Packed flush with no inter-container buffer to keep the canvas width sane; every other machine keeps the full 1-square buffer on all sides.
// Manufacturer row (rows B-D): Heavy Modular Frame, Computer, ACU (final) — left to right, matching production flow order.
// Circuit Board row (rows F-G): #1/#2 feed Computer at 106.67% (1 shard each); #3/#4 feed ACU at 66.67% (no shard, underclocked).
// Stator/Automated Wiring row (rows I-J): Stator's 10/min output splits into an exact 5/5 half to AW1 and AW2 — the only split in this network, matching the source dependency tree.
// Full shard total: 2 (Stator) + 2 (CB1/#2) + 0 (CB3/#4) + 0 (HMF) + 2+2 (AW1/#2) + 2 (Computer) + 2 (ACU) = 12 shards, same as the dependency tree.
// Belt tiers needed on individual storage-container outputs: Screw 240/min needs Mk.3 (270 cap); Cable to AW1/AW2 100/min each and Plastic to Computer 64/min and Wire 80/min need Mk.2 (120 cap); Steel Pipe to HMF 40/min, Steel Pipe to Stator 30/min, and the rest fit Mk.1 (60 cap) since each container now feeds only its one dedicated consumer.
// Revision note: earlier draft consolidated storage to 8 shared containers with manifold taps to save space; reverted per feedback that the dependency tree deliberately uses one dedicated container per edge (no splits except Stator) and that choice shouldn't have been made without asking — this version restores per-edge storage and keeps width down by dropping the buffer between storage containers only.
// Revision note: the standard below-grid INPUT strip was removed — every raw material's source/rate is already shown on its dedicated storage container in the storage row (rows M-N), so a duplicate external strip was redundant. The output strip above the grid is kept, since the final ACU has no on-grid box representing where it leaves to.
// Revision note: an earlier pass "corrected" HMF's Steel Pipe/Screw consumption to 30/184 per min based on a misremembered recipe; verified against satisfactory.wiki.gg and reverted — the real Heavy Modular Frame recipe (5 Modular Frame + 20 Steel Pipe + 5 Enc. Ind. Beam + 120 Screw per 30s craft) matches the original 40/240 figures exactly, so no change was needed there. Computer's Circuit Board/Cable/Plastic figures were separately verified against the wiki and are correct as-is (4 Circuit Board + 8 Cable + 16 Plastic per 24s craft, scaled to 160% clock = 16/32/64 per min).
